# Project Structure

A modern monorepo with clear separation between apps, shared packages, and development tools.

<Callout type="info">
  See **[Tech Stack](/docs/setup/tech-stack)** for technology details.
</Callout>

## Quick Overview

```
verifio/
├── apps/
│   ├── backend/
│   │   ├── auth/       # Authentication service
│   │   ├── verify/     # Email verification engine
│   │   ├── api-key/    # API key management
│   │   ├── credits/    # Credits and billing
│   │   ├── logs/       # Verification history
│   │   └── upload/     # File upload handling
│   └── frontend/
│       ├── web/        # Marketing website
│       ├── dashboard/  # User dashboard
│       └── docs/       # Documentation site
├── packages/           # Shared libraries
├── tools/              # Dev configs & utilities
└── local/              # Docker compose setup
```

## Packages

Shared libraries used across multiple apps:

| Package | Description |
|---------|-------------|
| `@verifio/db` | Database schema and Drizzle utilities |
| `@verifio/auth` | Authentication logic and session management |
| `@verifio/ui` | Design system and UI components |
| `@verifio/email` | Email templates and sending utilities |
| `@verifio/email-verify` | Core verification logic |
| `@verifio/cache` | Redis caching utilities |
| `@verifio/logger` | Logging utilities |
| `@verifio/api` | Shared API types and utilities |

## Tools

Configuration packages for consistent tooling:

| Package | Purpose |
|---------|---------|
| `@verifio/tailwind` | Shared Tailwind CSS configuration |
| `@verifio/tsconfig` | TypeScript configuration presets |

## Key Benefits

- **Shared code**: Reusable packages across all apps
- **Consistent tooling**: Unified configs and build setup
- **Clear boundaries**: Each service has a specific purpose
- **Easy scaling**: Add new services without affecting others
- **Type safety**: End-to-end TypeScript across the monorepo
