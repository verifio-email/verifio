# API Key Service

Manages API key generation, validation, and access control.

## Overview

| Property | Value |
|----------|-------|
| **Location** | `apps/backend/api-key` |
| **Framework** | ElysiaJS |
| **Port** | 8012 |
| **API Route** | `/api/api-key/*` |

## Starting the Service

```sh
bun run be:api-key:dev
```

## Features

- **Key generation** - Create new API keys with prefixes
- **Key validation** - Authenticate API requests
- **Permission scopes** - Read, write, admin access levels
- **Usage tracking** - Monitor key usage patterns
- **Key revocation** - Disable compromised keys

## API Key Format

```
vrf_live_xxxxxxxxxxxxxxxxxxxx
vrf_test_xxxxxxxxxxxxxxxxxxxx
```

- `vrf_` - Verifio prefix
- `live_` / `test_` - Environment indicator
- Random secure suffix

## Environment Variables

```sh
PORT=8012
PG_URL=postgresql://verifio:verifio123@localhost:5432/verifio
```

## Database Tables

- `api_key` - API key storage and metadata
- Organization-scoped key management

<Callout type="warn">
  API keys are hashed before storage. The full key is only shown once at creation.
</Callout>
