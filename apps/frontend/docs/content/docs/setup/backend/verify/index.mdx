# Verify Service

The core email verification engine that performs all validation checks.

## Overview

| Property | Value |
|----------|-------|
| **Location** | `apps/backend/verify` |
| **Framework** | ElysiaJS |
| **Port** | 3040 |
| **API Route** | `/api/verify/*` |

## Starting the Service

```sh
bun run be:verify:dev
```

## Verification Checks

The verify service performs multiple validation layers:

| Check | Description |
|-------|-------------|
| **Syntax** | RFC 5322 email format validation |
| **DNS** | Domain existence and record lookup |
| **MX** | Mail server configuration check |
| **SMTP** | Mailbox verification via SMTP handshake |
| **Disposable** | Temporary email provider detection |
| **Role-based** | Generic address detection (info@, support@) |
| **Catch-all** | Server catch-all configuration detection |

## Verification States

```typescript
type State = 'deliverable' | 'undeliverable' | 'risky' | 'unknown'
```

## Environment Variables

```sh
PORT=3040
PG_URL=postgresql://verifio:verifio123@localhost:5432/verifio
```

## Related Services

- **Credits** - Deducts credits per verification
- **Logs** - Records verification history
- **Workflow** - Called for bulk verification jobs

<Callout type="warn">
  The verify service requires network access for DNS and SMTP checks.
</Callout>
