## Overview

The Verifio API provides RESTful endpoints for email verification. All API requests require authentication using an API key.

## Base URL

```
https://verifio.email/api/verify
```

## Authentication

Include your API key in the `x-api-key` header:

```bash
curl -H "x-api-key: your_api_key" https://verifio.email/api/verify/v1/email
```

## Available Endpoints

### Single Email Verification

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/v1/email` | Verify a single email address |
| `POST` | `/v1/verify` | Alternative endpoint for single verification |

### Bulk Verification

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/v1/bulk` | Start a bulk verification job |
| `POST` | `/v1/bulk-verify` | Alternative bulk verification endpoint |
| `GET` | `/v1/bulk-jobs/:jobId` | Get bulk job status |
| `GET` | `/v1/bulk-jobs/:jobId/results` | Get bulk job results |

### Jobs & Results

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/v1/jobs` | List all verification jobs |
| `GET` | `/v1/jobs/:jobId` | Get a specific job |
| `GET` | `/v1/jobs/:jobId/results` | Get job results |
| `GET` | `/v1/results/:resultId` | Get a specific result |
| `GET` | `/v1/history` | Get verification history |

### Health

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/` | Service health and status |
| `GET` | `/health/postgres` | Database health check |

## Response Format

All responses are returned in JSON format:

```json
{
  "email": "user@example.com",
  "state": "deliverable",
  "score": 95,
  "reason": "Email is valid and deliverable",
  "checks": {
    "syntax": true,
    "dns": true,
    "disposable": false,
    "role": false,
    "smtp": true
  }
}
```

## Rate Limits

API requests are rate-limited based on your plan. See your dashboard for current limits.
