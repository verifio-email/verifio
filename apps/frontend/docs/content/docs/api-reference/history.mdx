The History API allows you to retrieve past verification results without re-verifying emails.

## Get History

**GET** `/api/v1/history`

Get a paginated list of past email verifications. ðŸ”’ Authentication Required

### Query Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `page` | number | 1 | Page number for pagination. |
| `limit` | number | 20 | Number of results per page. Maximum 100. |
| `state` | string | - | Filter by state: `deliverable`, `undeliverable`, `risky`, `unknown`. |
| `from` | string | - | Start date in ISO 8601 format (e.g., `2024-01-01`). |
| `to` | string | - | End date in ISO 8601 format. |
| `search` | string | - | Search emails containing this string. |

## Example Request

### cURL

```bash
curl "https://verifio.email/api/v1/history?page=1&limit=20&state=deliverable" \
  -H "Authorization: Bearer vrf_your_api_key"
```

### Node.js

```typescript
const history = await verifio.history.list({
  page: 1,
  limit: 20,
  state: 'deliverable',
  from: '2024-01-01',
  to: '2024-01-31'
});

console.log(`Total: ${history.pagination.total}`);

history.items.forEach(result => {
  console.log(result.email, result.state, result.verifiedAt);
});
```

## Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique identifier for the verification. |
| `email` | string | The verified email address. |
| `state` | string | Verification result: `deliverable`, `undeliverable`, `risky`, or `unknown`. |
| `score` | number | Quality score from 0-100. |
| `reason` | string | Machine-readable reason code. |
| `verifiedAt` | string | ISO 8601 timestamp of when the verification occurred. |
| `source` | string | Source of verification: `api` (single), `bulk`, or `playground`. |

## Get Single Verification Detail

**GET** `/api/v1/history/{verificationId}`

Get full details of a specific verification. ðŸ”’ Authentication Required

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `verificationId` | string | Yes | The verification ID. |

### Example

```bash
curl https://verifio.email/api/v1/history/ver_123 \
  -H "Authorization: Bearer vrf_your_api_key"
```

Returns the full verification result with all checks, same as the single verification endpoint response.

## Use Cases

### Check if Email was Previously Verified

```typescript
const history = await verifio.history.list({
  search: 'user@example.com',
  limit: 1
});

if (history.items.length > 0) {
  const cached = history.items[0];
  console.log(`Already verified: ${cached.state} (${cached.verifiedAt})`);
} else {
  // Verify now
  const result = await verifio.verify('user@example.com');
}
```

## Data Retention

Verification history is retained for **90 days**. After this period, records are automatically deleted.
