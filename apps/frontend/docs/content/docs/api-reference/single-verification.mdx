Verify a single email address and receive immediate results.

## Endpoint

**POST** `/api/v1/verify`

Verify a single email address and receive detailed validation results.

üîí **Authentication Required**

## Request Body

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `email` | string | Yes | - | The email address to verify. |
| `skipDisposable` | boolean | No | false | Skip the disposable email check. |
| `skipRole` | boolean | No | false | Skip the role-based email check. |
| `skipTypo` | boolean | No | false | Skip typo detection and suggestions. |

## Example Request

### cURL

```bash
curl -X POST https://verifio.email/api/v1/verify \
  -H "Authorization: Bearer vrf_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "hello@example.com"
  }'
```

### Node.js

```typescript
import { Verifio } from '@verifio/email-verification';

const verifio = new Verifio({ apiKey: 'vrf_your_api_key' });

const result = await verifio.verify('hello@example.com');
console.log(result);
```

## Response Fields

### Top-Level Fields

| Field | Type | Description |
|-------|------|-------------|
| `email` | string | The email address that was verified. |
| `state` | string | Primary verdict: `deliverable`, `undeliverable`, `risky`, or `unknown`. |
| `score` | number | Quality score from 0-100. Higher is better. |
| `reason` | string | Machine-readable reason code explaining the state. |

### Checks Object

| Field | Type | Description |
|-------|------|-------------|
| `checks.syntax.valid` | boolean | Whether the email has valid syntax. |
| `checks.dns.valid` | boolean | Whether the domain has valid DNS records. |
| `checks.dns.hasMx` | boolean | Whether the domain has MX records. |
| `checks.disposable.isDisposable` | boolean | Whether the email is from a disposable provider. |
| `checks.role.isRole` | boolean | Whether the email is a role-based address. |
| `checks.smtp.mailboxExists` | boolean | Whether the mailbox exists on the server. |
| `checks.smtp.catchAll` | boolean | Whether the domain accepts all emails (catch-all). |

## State Values

| State | Description | Action |

## Reason Codes

| Reason | Description |
|--------|-------------|
| `valid_mailbox` | Mailbox exists and accepts mail |
| `invalid_syntax` | Email format is invalid |
| `invalid_domain` | Domain doesn't exist |
| `no_mx_records` | Domain has no mail server |
| `mailbox_not_found` | Mailbox doesn't exist |
| `disposable_email` | Temporary/disposable email |
| `role_based_email` | Generic role address |
| `catch_all` | Domain accepts all addresses |
| `smtp_timeout` | Mail server didn't respond |
