The Verifio API provides a RESTful interface for email verification. All endpoints return JSON responses and require authentication.

## Base URL

```
https://verifio.email/api/v1
```

## Authentication

All API requests require an API key passed in the `Authorization` header:

```bash
Authorization: Bearer vrf_your_api_key
```

See [Authentication](/getting-started/authentication) for details.

## Endpoints Overview

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/verify` | POST | Verify a single email |
| `/bulk/upload` | POST | Upload emails for bulk verification |
| `/bulk/{jobId}` | GET | Get bulk job status |
| `/bulk/{jobId}/results` | GET | Get bulk job results |
| `/history` | GET | Get verification history |

## Response Format

All responses follow this structure:

```json
{
  "success": true,
  "data": { ... }
}
```

Error responses:

```json
{
  "success": false,
  "error": {
    "code": "error_code",
    "message": "Human readable message"
  }
}
```

## Common Error Codes

| Code | Status | Description |
|------|--------|-------------|
| `unauthorized` | 401 | Invalid or missing API key |
| `insufficient_credits` | 402 | Not enough credits |
| `rate_limit_exceeded` | 429 | Too many requests |
| `validation_error` | 400 | Invalid request parameters |
| `not_found` | 404 | Resource not found |
| `internal_error` | 500 | Server error |

## Credits

Each verification consumes credits:

| Operation | Credits |
|-----------|---------|
| Single verification | 1 credit |
| Bulk verification | 1 credit per email |
