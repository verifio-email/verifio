---
title: Bulk Verification
description: Verify thousands of emails in background jobs.
---

Bulk verification allows you to verify large lists of emails asynchronously. Upload your emails, and Verifio processes them in the background.

## Workflow

1. **Upload Emails** - POST a list of emails to `/api/v1/bulk/upload`. Receive a job ID.
2. **Poll for Status** - GET `/api/v1/bulk/{jobId}` to check progress. Repeat until status is `completed`.
3. **Get Results** - GET `/api/v1/bulk/{jobId}/results` to retrieve verification results.

## Upload Emails

**POST** `/api/v1/bulk/upload`

Upload a list of emails for bulk verification. ðŸ”’ Authentication Required

### Request Body

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `emails` | string[] | Yes | Array of email addresses to verify. Maximum 10,000 per request. |
| `name` | string | No | Optional name for the job to identify it later. |

### cURL Example

```bash
curl -X POST https://verifio.email/api/v1/bulk/upload \
  -H "Authorization: Bearer vrf_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "emails": [
      "user1@example.com",
      "user2@example.com",
      "user3@example.com"
    ],
    "name": "Marketing List Q1"
  }'
```

### Node.js Example

```typescript
const job = await verifio.bulk.verify([
  'user1@example.com',
  'user2@example.com',
  'user3@example.com'
]);

console.log(job.id);     // "job_abc123"
console.log(job.status); // "pending"
```

### Response

```json
{
  "success": true,
  "data": {
    "id": "job_abc123",
    "status": "pending",
    "totalEmails": 3,
    "processedEmails": 0,
    "createdAt": "2024-01-15T10:30:00Z"
  }
}
```

## Check Job Status

**GET** `/api/v1/bulk/{jobId}`

Get the current status of a bulk verification job. ðŸ”’ Authentication Required

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `jobId` | string | Yes | The job ID returned from the upload endpoint. |

### Example Request

```bash
curl https://verifio.email/api/v1/bulk/job_abc123 \
  -H "Authorization: Bearer vrf_your_api_key"
```

### Response

```json
{
  "success": true,
  "data": {
    "id": "job_abc123",
    "name": "Marketing List Q1",
    "status": "processing",
    "totalEmails": 1000,
    "processedEmails": 450,
    "progress": 45,
    "stats": {
      "deliverable": 380,
      "undeliverable": 45,
      "risky": 20,
      "unknown": 5
    },
    "createdAt": "2024-01-15T10:30:00Z",
    "updatedAt": "2024-01-15T10:35:00Z"
```

### Job Status Values

| Status | Description |
|--------|-------------|
| `pending` | Job is queued for processing |
| `processing` | Verification in progress |
| `completed` | All emails verified |
| `failed` | Job failed (check error message) |

## Get Results

**GET** `/api/v1/bulk/{jobId}/results`

Get verification results for a completed job. ðŸ”’ Authentication Required

### Query Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `page` | number | 1 | Page number for pagination. |
| `limit` | number | 50 | Number of results per page. Maximum 100. |
| `state` | string | - | Filter by state: `deliverable`, `undeliverable`, `risky`, `unknown`. |

### Example Request

```bash
curl "https://verifio.email/api/v1/bulk/job_abc123/results?page=1&limit=50" \
  -H "Authorization: Bearer vrf_your_api_key"
```

## Polling Best Practices

> [!TIP]
> Use exponential backoff when polling for job status to avoid rate limits.

```typescript
async function waitForJob(jobId: string): Promise<BulkJob> {
  let delay = 1000; // Start with 1 second
  
  while (true) {
    const job = await verifio.bulk.getJob(jobId);
    
    if (job.status === 'completed' || job.status === 'failed') {
      return job;
    }
    
    await new Promise(resolve => setTimeout(resolve, delay));
    delay = Math.min(delay * 1.5, 30000); // Max 30 seconds
  }
}
```

## Limits

| Limit | Value |
|-------|-------|
| Emails per upload | 10,000 |
| Concurrent jobs | 5 |
| Job retention | 30 days |

## Credit Usage

Each email in a bulk job consumes **1 credit**. Credits are deducted when the job starts processing, not when uploaded.
