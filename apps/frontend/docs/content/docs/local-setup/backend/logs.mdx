Centralized logging for audit trails, debugging, and analytics.

## Quick Start

| Property | Value |
|----------|-------|
| **Location** | `apps/backend/logs` |
| **Port** | 8003 |
| **Start Command** | `bun run be:logs:dev` |

## Environment Variables

```bash
PORT=8003
BASE_URL=https://local.verifio.email
PG_URL=postgresql://verifio:verifio123@localhost:5432/verifio
REDIS_URL=redis://:verifio123@localhost:6379
```

## Main Endpoints

| Route | Method | Description | Auth Required |
|-------|--------|-------------|---------------|
| `/api/logs/v1/add` | POST | Add log entry | Session |
| `/api/logs/v1/query` | GET | Query logs | Session |
| `/api/logs/health` | GET | Health check | No |

## Log Entry Schema

| Field | Type | Description |
|-------|------|-------------|
| `userId` | string | User who performed action |
| `organizationId` | string | Organization context |
| `service` | string | Service name |
| `endpoint` | string | API endpoint |
| `status` | string | "success" or "error" |
| `creditsUsed` | number | Credits consumed |
| `ipAddress` | string | Client IP |

## Querying Logs

```bash
curl "https://api.verifio.email/api/logs/v1/query?service=verify&status=error&limit=50" \
  --cookie-jar cookies.txt --cookie cookies.txt
```

## Database Tables

- `activityLogs` - All activity entries

## Common Issues

**Logs not appearing:**
- Verify Logs service is running
- Check other services have correct `BASE_URL`
- Look for errors in service logs

**Slow queries:**
- Add filters (service, status, date range)
- Reduce `limit` parameter
