# Tools Service

Provides free email verification tools that don't require authentication or credits. Perfect for testing and basic validation.

## Overview

| Property | Value |
|----------|-------|
| **Location** | `apps/backend/tools` |
| **Framework** | ElysiaJS |
| **Port** | 8006 |
| **Route Prefix** | `/api/tools` |
| **Request Timeout** | 30 seconds |

## What It Does

The Tools service provides:

- **Syntax Validation** - RFC 5322 email format checking
- **Disposable Detection** - Identify temporary email services
- **Deliverability Check** - Basic domain and MX verification
- **No Authentication** - Public endpoints for easy testing
- **Rate Limiting** - Per-IP limits to prevent abuse
- **Free to Use** - No credits required

## Starting the Service

```bash
bun run be:tools:dev
```

## Environment Variables

Create a `.env` file in `apps/backend/tools/`:


## Available Tools

### 1. Syntax Validation

Validates email format according to RFC 5322 standards.

**Request:**
```bash
curl -X POST https://local.verifio.email/api/tools/v1/syntax/validate \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com"}'
```

**Response:**
```json
{
  "valid": true,
  "email": "user@example.com",
  "normalized": "user@example.com",
  "issues": []
}
```

**What it checks:**
- Valid email format
- Correct syntax structure
- Common typos (e.g., `@gmial.com` → `@gmail.com`)
- Provider detection (Gmail, Outlook, etc.)

### 2. Disposable Detection

Checks if the email domain is from a disposable email service.

**Request:**
```bash
curl -X POST https://local.verifio.email/api/tools/v1/disposable/check \
  -H "Content-Type: application/json" \
  -d '{"email": "user@tempmail.com"}'
```

**Response:**
```json
{
  "disposable": true,
  "domain": "tempmail.com",
  "provider": "TempMail",
  "trustScore": 0.1
}
```

**What it checks:**
- Known disposable email domains
- Temporary email services
- 10-minute email services
- Trust score based on domain reputation

### 3. Deliverability Check

Basic check for email deliverability without full SMTP verification.

**Request:**
```bash
curl -X POST https://local.verifio.email/api/tools/v1/deliverability/check \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com"}'
```

**Response:**
```json
{
  "deliverable": true,
  "domain": "example.com",
  "hasMXRecords": true,
  "mxRecords": ["mail.example.com"],
  "canDeliver": true
}
```

**What it checks:**
- Domain exists
- MX records are present
- Mail servers are configured
- Basic DNS configuration

<Callout type="warn">
  The deliverability check does **not** perform SMTP handshakes. For full verification, use the **Verify Service**.
</Callout>

## Rate Limiting

All public endpoints are rate limited to prevent abuse:

### How It Works

- **Per-IP Limits**: Each IP address has independent limits
- **Redis-Backed**: Distributed rate limiting across multiple instances
- **Sliding Window**: Accurate rate limiting using Redis
- **Automatic Cleanup**: Old entries are automatically removed

### Rate Limit Response

When limit is exceeded:

```json
{
  "success": false,
  "error": "Rate limit exceeded",
  "data": {
    "limit": 100,
    "remaining": 0,
    "resetAt": "2024-01-30T12:00:00Z"
  }
}
```

## Use Cases

### 1. Email Input Validation

Validate emails in real-time as users type:

```typescript
// Frontend example
async function validateEmailInput(email: string) {
  const response = await fetch('/api/tools/v1/syntax/validate', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ email }),
  });

  return await response.json();

```

### 2. Form Pre-submission Checks

Check for disposable emails before form submission:

```typescript
async function checkDisposable(email: string) {
  const response = await fetch('/api/tools/v1/disposable/check', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ email }),
  });

  const { disposable } = await response.json();

  if (disposable) {
    alert('Please use a permanent email address');
  }
}
```

### 3. Quick Domain Verification

Verify a domain has MX records without full verification:

```typescript
async function quickDomainCheck(email: string) {
  const response = await fetch('/api/tools/v1/deliverability/check', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ email }),
  });

  const { canDeliver } = await response.json();

  return canDeliver;
}
```

## Troubleshooting

### Service won't start

- Check port 8006 is available: `lsof -i :8006`
- Verify Redis is running: `docker ps | grep redis`
- Check `.env` file exists and is correct

### "Rate limit exceeded" error

- Wait for the time window to reset (default 60 seconds)
- Reduce request frequency
- Consider implementing client-side caching

### Redis connection errors

```bash
# Test Redis connection
redis-cli -h localhost -p 6379 -a verifio123 ping
```

- Verify `REDIS_URL` in `.env` matches Docker configuration
- Check Redis container is running: `docker ps | grep redis`

### Tools returning incorrect results

- Verify the **@verifio/email-verify** package is up to date
- Check for typos in the email address
- Some disposable domains may not be detected (new services)

### Health check showing services as down

- Verify all services are running: `bun run verifio:dev`
- Check service ports are correct
- Verify network connectivity between services

## Performance Considerations

- **No Authentication**: Faster response times (no auth overhead)
- **Redis Caching**: Rate limits checked in Redis (fast)
- **Lightweight Checks**: No SMTP connections (faster than full verification)
- **Rate Limited**: Prevents abuse while maintaining availability

## Security Considerations

### Rate Limiting

- **Per-IP Limits**: Prevents single-source abuse
- **Redis-Backed**: Distributed rate limiting
- **Configurable**: Adjust limits based on load

### Input Validation

- **Email Validation**: All inputs are validated
- **Length Limits**: Email addresses limited to 254 characters
- **Domain Validation**: Domains are validated before processing

### No Sensitive Data

- **No PII Storage**: No emails are stored
- **No User Tracking**: No user accounts or sessions
- **No Logs**: Minimal logging for privacy
