# Verify Single Email API

Real-time email verification for user signups, form submissions, and validation workflows.

## Quick Reference

| Property | Value |
|----------|-------|
| **Endpoint** | `POST /api/verify/v1/email` |
| **Rate Limit** | 2 req/min (unauthenticated), plan-based (authenticated) |
| **Authentication** | API Key (Bearer token) |
| **Cost** | 1 credit per verification |

## Response Structure

### Success Response

```json
{
  "success": true,
  "data": {
    "email": "test@gmail.com",
    "state": "deliverable",
    "reason": "valid_mailbox",
    "score": 95,
    "checks": { /* detailed checks */ },
    "analytics": { /* insights */ }
  }
}
```

### Core Fields

| Field | Type | Description |
|-------|------|-------------|
| `email` | string | Verified email address |
| `state` | string | `deliverable`, `undeliverable`, `risky`, `unknown` |
| `reason` | string | Specific reason code |
| `score` | number | Quality score (0-100) |

### State Interpretation

| State | Meaning | Action |
|-------|---------|--------|
| `deliverable` | ‚úÖ Valid, will receive mail | Safe to send |
| `undeliverable` | ‚ùå Invalid or doesn't exist | Block email |
| `risky` | ‚ö†Ô∏è Valid but has concerns | Use caution |
| `unknown` | ‚ùì Cannot determine | Retry or manual review |

### Score Guide

| Range | Quality | Recommendation |
|-------|---------|----------------|
| 90-100 | Excellent | ‚úÖ High deliverability |
| 70-89 | Good | ‚úÖ Generally safe |
| 50-69 | Fair | ‚ö†Ô∏è Monitor bounce rates |
| 30-49 | Poor | ‚ö†Ô∏è High risk |
| 0-29 | Very Poor | ‚ùå Do not send |

## Detailed Checks

### Available Checks

```json
{
  "checks": {
    "syntax": { "valid": true },
    "dns": { "valid": true, "hasMx": true, "mxRecords": [...] },
    "disposable": { "isDisposable": false },
    "role": { "isRole": false },
    "freeProvider": { "isFree": true, "provider": "gmail" },
    "typo": { "hasTypo": false, "suggestion": null },
    "smtp": { "valid": true, "mailboxExists": true, "isCatchAll": false }
  }
}
```

| Check | Purpose |
|-------|---------|
| `syntax` | Email format validation |
| `dns` | Domain existence and MX records |
| `disposable` | Temporary email detection |
| `role` | Generic addresses (info@, support@) |
| `freeProvider` | Free email services (Gmail, Yahoo) |
| `typo` | Common typo detection with suggestions |
| `smtp` | Mailbox verification via SMTP |

## Request Options

Optionally skip specific checks to speed up verification or customize behavior based on your requirements.

Customize verification behavior:

```json
{
  "email": "user@example.com",
  "options": {
    "skipDisposable": false,
    "skipRole": false,
    "skipTypo": false
  }
}
```

<Callout>
  Skipping checks speeds up verification but reduces comprehensiveness. Only skip if you have specific requirements.
</Callout>

## Error Handling

### HTTP Status Codes

| Code | Meaning | Action |
|------|---------|--------|
| 200 | Success | Parse response |
| 400 | Bad Request | Check request format |
| 401 | Unauthorized | Verify API key |
| 402 | Payment Required | Add credits |
| 429 | Rate Limited | Implement retry with backoff |
| 500 | Server Error | Retry after delay |

### Error Response

```json
{
  "success": false,
  "error": "Description of error",
  "retryAfter": 60
}
```

## Best Practices

### 1. Secure API Key Storage

Use environment variables, never hardcode:

```javascript
// .env file
VERIFIO_API_KEY=vrf_sk_live_xxx

// Code
const verifio = new Verifio({ apiKey: process.env.VERIFIO_API_KEY });
```

### 2. Implement Retry Logic

```javascript
async function verifyWithRetry(email, maxRetries = 3) {
  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      return await verifio.verify(email);
    } catch (error) {
      if (error.status === 429 || error.status >= 500) {
        const waitTime = Math.pow(2, attempt) * 1000; // Exponential backoff
        await new Promise(resolve => setTimeout(resolve, waitTime));
      } else {
        throw error;
      }
    }
  }
  throw new Error('Max retries exceeded');
}
```

### 3. Use Typo Suggestions

```javascript
if (result.checks.typo.hasTypo) {
  const suggestion = result.checks.typo.suggestion;
  // Prompt user: "Did you mean {suggestion}?"
}
```

### 4. Handle All States

```javascript
switch (result.state) {
  case 'deliverable':
    // Proceed with email
    break;
  case 'undeliverable':
    // Show error to user
    break;
  case 'risky':
    // Warn user or flag for review
    break;
  case 'unknown':
    // Retry later or manual review
    break;
}
```

## Common Use Cases

### Form Validation

```javascript
app.post('/signup', async (req, res) => {
  const { email } = req.body;
  const verification = await verifio.verify(email);
  
  if (verification.state === 'undeliverable') {
    return res.status(400).json({
      error: 'Invalid email address',
      suggestion: verification.checks.typo.suggestion
    });
  }
  
  if (verification.checks.disposable.isDisposable) {
    return res.status(400).json({
      error: 'Please use a permanent email address'
    });
  }
  
  // Proceed with signup
});
```

### CRM Data Cleaning

```javascript
for (const contact of contacts) {
  const result = await verifio.verify(contact.email);
  
  await updateContact(contact.id, {
    emailVerified: result.state === 'deliverable',
    emailScore: result.score,
    riskLevel: result.analytics.riskLevel
  });
}
```

## Common Reason Codes

### Deliverable
- `valid_mailbox` ‚Äî Mailbox exists and accepts mail
- `accepted_email` ‚Äî SMTP server accepted

### Undeliverable
- `invalid_syntax` ‚Äî Malformed email
- `invalid_domain` ‚Äî Domain doesn't exist
- `no_mx_records` ‚Äî No mail servers configured
- `mailbox_not_found` ‚Äî Mailbox doesn't exist
- `mailbox_full` ‚Äî Mailbox is full

### Risky
- `disposable_email` ‚Äî Temporary email service
- `role_based_email` ‚Äî Generic address (info@, support@)
- `catch_all_domain` ‚Äî Accepts all addresses
- `free_email_provider` ‚Äî Free service (Gmail, Yahoo)

### Unknown
- `timeout` ‚Äî Server timeout
- `connection_error` ‚Äî Network issue
- `unknown_error` ‚Äî Unexpected error
